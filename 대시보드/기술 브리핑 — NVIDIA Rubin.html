아키텍처 NVIDIA GTC 2026 · Rubin 플랫폼 · 2026-07-23 정리

NVIDIA Rubin 플랫폼 — HBM4 288GB·Vera 커스텀 CPU·LPDDR5X. 삼성·SK엔 HBM+LPDDR 이중 수요, 단 CPU 성능 주장은 걸러 볼 것

Rubin GPU는 듀얼다이 1패키지(NV-HBI)·HBM4·NVLink 6로, 새 Vera CPU는 88개 커스텀 Arm 코어(Olympus)에 SOCAMM2 LPDDR5X를 대량 얹는다. 우리 관점의 핵심 = HBM4 퀄 경쟁 + 데이터센터 LPDDR이라는 두 번째 매출축. 단 NVIDIA의 "x86 대비 1.8배" CPU 주장은 코어당 수치라 종합 성능과 다르다.

우리 영향 — 먼저 볼 것

호재
HBM 수요 (SK하이닉스·삼성전자·Micron) — GPU당 HBM4 288GB(대역폭 22TB/s) + Vera CPU가 LPDDR5X 256GB~1.5TB 대량 탑재. 대당 메모리 소비 구조적 확대.
호재
데이터센터 LPDDR 신카테고리 — SOCAMM2 모듈·9600MT/s·ECC LPDDR5X = 서버 LPDDR 신규 격전지. HBM에 이은 두 번째 축, 3사 경쟁 재점화.
관건
HBM4 · SOCAMM2 양산 퀄 리더십 — 22TB/s·NV-HBI 듀얼다이·9600MT/s 등급 = 속도·수율 난이도 급등. 최초 대량 퀄 통과사가 초도 물량 독식(SK 선두, 삼성 추격).
관건
x86 잠식 — Vera 88 커스텀 Arm(Olympus)이 AI 랙서 Intel·AMD CPU 대체. 서버 CPU 지형 이동 → 우리 메모리·패키징 믹스에 파급.
리스크
퀄 미달 시 — 고난도 스펙 못 맞추면 초도 배정 탈락, 다음 리비전까지 매출 공백. 스펙 상향이 진입장벽으로 작동.

NVIDIA 주장 vs 검증 — 사실과 판단

사실
위 스펙(HBM4 288GB·22TB/s·NVLink6·Vera 88코어·LPDDR5X 1.2TB/s·NVLink-C2C 1.8TB/s) = 백서·블로그 명시.
판단
"에이전틱 1.8배 vs x86" = 코어당·풀로드 SPEC 점수. 종합 처리량 아님.
판단
종합 실측(SPECrate 2026_int)은 Vera 925 vs AMD Turin(128코어) 898 ≈ 사실상 무승부(1.03×). Vera 우위는 코어당 효율이지 소켓 총량 아님 — CPU 성능 과대해석 주의.
판단
"에너지효율 10배(GPU)" = 내부 2T MoE 워크로드 Pareto 종합치. 단일 하드웨어 스펙 아님.
판단
"BF16 4배·FP32 2배" = exponential 연산 한정. 범용 compute 배수 아님.
판단
클록·공정노드·Grace 정량 비교치는 양쪽 소스 모두 미공개.
