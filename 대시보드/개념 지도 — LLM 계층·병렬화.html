AI와 그 인프라 구조 — 학습·추론에서 계층·병렬화, GPU 랙까지 한 장에

출발점은 두 국면 — 학습(Training)과 추론(Inference, 다시 Prefill·Decode). 국면마다 병목이 다르고, 그걸 푸는 게 1부: AI 모델을 나누는 기법(GQA·MoE·어텐션 계보)과 2부: 인프라에 태우는 방식(병렬·스케일업/아웃·GPU 랙)이다.

큰 그림 — 학습 vs 추론, Prefill vs Decode

AI 모델을 쓰는 방식은 두 국면. 추론은 다시 두 단계로 쪼개진다. 어느 국면·단계냐가 병목과 핵심 기법을 정한다 — 아래 모든 내용의 출발점.

AI 모델 사용
compute-bound학습 (Training) — 순전파+역전파, 모든 토큰 병렬
추론 (Inference) — 두 단계로 쪼개짐

bandwidth-boundDecode — 토큰 1개씩 자기회귀, KV Cache 읽음

Prefill은 GPU를 꽉 채우는 연산, Decode는 대역폭에 굶주리는 순차 연산 → KV 절감(GQA)·스케일업(NVL72)·Prefill 전용칩(CPX)이 갈린다. 그래서 추론을 Prefill/Decode로 분리(disaggregation)해 서로 다른 하드웨어에 태운다.

3 · 블록별 나누기 기법 — 병렬 배치 비교

1개 레이어 안에서 어느 블록을 어떤 단위로 나누는가. 구조적 = 메모리·연산 절감(모델 레벨), 병렬 = 여러 GPU 분산(인프라 레벨, 2부에서 상술).

① Attention Block

Head 묶음 방식으로 나눔

MHA / GQA 구조적
MHA: Query·Key·Value Head를 1:1:1 보유. GQA: 여러 Query Head가 1개 K/V Head 그룹을 공유 → KV Cache 메모리 절감.
TP · Tensor Parallel 병렬
하나의 연산을 쪼개 여러 GPU가 동시 수행. Attention은 Head 단위로 분배 (예: 32 Head → 4 GPU가 8개씩).

② MLP / FFN Block

Expert 분할 방식으로 나눔

Dense 기존
모든 토큰이 단 하나의 거대한 MLP 가중치를 통과.
MoE · Mixture of Experts 구조적
MLP를 통째로 여러 개 복사해 독립 Expert로 분할. Router가 토큰별 1~2개만 선택 → 파라미터 용량↑, 실제 FLOPs는 절감.
EP · Expert Parallel 병렬
MoE의 각 Expert를 서로 다른 GPU에 분산 배치하는 병렬 기법.

4 · ① Attention 심화 — 어텐션 계보 (MHA → 선형)

5 · 병렬화 완전체 — 5D 병렬

3번의 TP·EP는 병렬의 일부일 뿐. 학습과 대규모 추론은 아래 5개 축을 겹쳐서 쓴다.

병렬나누는 축통신 패턴주 무대

프론티어 학습은 이 5D를 동시에 겹쳐 씀 — 예: MoE 세계기록(GB300 NVL72)엔 DP·TP·PP·EP가 한꺼번에 들어간다.

6 · 물리 계층 — GPU → 랙 → 클러스터, 스케일업 vs 스케일아웃

병렬 기법은 결국 어떤 하드웨어 계층 위에 올리느냐로 갈린다. NVLink로 묶인 랙(NVL72) 안은 스케일업, 랙과 랙 사이는 스케일아웃이다.

AI 팩토리 (데이터센터) — Pod 여러 개
스케일아웃Pod / 클러스터 — 랙 여러 개, Ethernet·InfiniBand 연결
스케일업 도메인Rack = NVL72 — 72 Rubin GPU + 36 Vera CPU, NVLink로 한 덩어리
└ Compute Tray — Vera CPU 1 + Rubin GPU 2 (NVLink-C2C, 액냉)
└ GPU (Rubin) — HBM4 288GB · 듀얼다이 1패키지 · NVLink 6 (3.6TB/s)

이 경계선(NVLink 랙 안 vs 밖)이 곧 병렬 기법이 갈리는 선 — 통신 잦은 TP·EP는 랙 안(스케일업), 덜한 DP·PP는 랙 밖(스케일아웃). GPU 자체 내부 구조는 GPU 구조 시뮬레이터 ↗.

스케일업 (Scale-up)

노드 내부 · 초고대역 링크로 GPU를 한 덩어리로

NVLink / NVSwitch · NVL72
Rubin NVLink 6 = 3.6TB/s. 통신 잦은 TP·EP가 여기. "대역폭 벽"을 노드 안에서 해결.

스케일아웃 (Scale-out)

노드 간 · 이더넷/InfiniBand로 대수를 늘림

Spectrum-X · InfiniBand · RNG
통신 덜한 DP·PP가 여기. 클러스터 규모로 확장. 아마존 RNG 토폴로지처럼 오버서브스크립션·비용이 관건.

매핑: TP → 스케일업 · EP → 스케일업 우선 · DP·PP → 스케일아웃 · SP → 둘 다.

7 · 정밀도·양자화 — FP16 → FP4

비트를 줄일수록 메모리·대역폭·용량↓, 처리량↑. Decode(대역폭 병목)에 특히 효과. 학습은 BF16/FP8 혼합, 추론은 FP8/FP4 양자화가 표준.

FP32 기준→ BF16 학습 표준→ FP8 Hopper~→ FP4 NVFP4·MXFP · Blackwell·Rubin

품질 손실은 블록 스케일링(MXFP)·LUT 압축으로 방어. 단 비트를 줄여도 파라미터 용량은 남는다 — Kimi K3 2.8T는 FP4로 눌러도 단일 B200에 안 들어가 NVL72급이 필요하다. Rubin은 NVFP4로 50PFLOPS, LUT-B 압축까지 얹었다.

★ 종합 한 장 — AI 모델 × 인프라

1부·2부가 이 표 한 장으로 수렴한다. 4개 축이 직교 — 어느 블록(Attn/MLP) · 유형(구조적/병렬) · 핵심 단계(학습/Prefill/Decode) · 어느 네트워크(스케일업/아웃).
