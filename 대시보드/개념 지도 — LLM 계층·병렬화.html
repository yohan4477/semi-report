Transformer Layer 1개 안에서 "어느 블록을, 어떤 단위로 나누느냐"가 GQA·TP·MoE·EP를 가른다

한 레이어는 Attention(토큰 간 맥락)과 MLP(토큰 내부 가공) 두 블록으로 나뉜다. 이 둘을 각각 어떻게 쪼개느냐에 따라 메모리를 아끼는 구조적 기법(GQA·MoE)과 GPU에 흩뿌리는 병렬 기법(TP·EP)이 갈린다. 아래는 그 지도 + 최근 우리 신호가 이 개념 어디에 붙는지.

3 · 블록별 나누기 기법 — 병렬 배치 비교

1개 레이어 안에서 어느 블록을 어떤 단위로 나누는가에 따라 기술이 갈린다. 구조적 = 메모리·연산 절감, 병렬 = 여러 GPU 분산.

① Attention Block

Head 묶음 방식으로 나눔

MHA / GQA 구조적
MHA: Query·Key·Value Head를 1:1:1 보유. GQA: 여러 Query Head가 1개 K/V Head 그룹을 공유 → KV Cache 메모리 절감.
TP · Tensor Parallel 병렬
하나의 연산을 쪼개 여러 GPU가 동시 수행. Attention은 Head 단위로 분배 (예: 32 Head → 4 GPU가 8개씩).

② MLP / FFN Block

Expert 분할 방식으로 나눔

Dense 기존
모든 토큰이 단 하나의 거대한 MLP 가중치를 통과.
MoE · Mixture of Experts 구조적
MLP를 통째로 여러 개 복사해 독립 Expert로 분할. Router가 토큰별 1~2개만 선택 → 파라미터 용량↑, 실제 FLOPs는 절감.
EP · Expert Parallel 병렬
MoE의 각 Expert를 서로 다른 GPU에 분산 배치하는 병렬 기법.

5 · 실제 신호에서 — 이 개념이 뉴스에 나타나는 곳

MoE
MI500X TDM MoE 디스크립터 공유 방식 — AMD가 전 Expert 공유 descriptor 도입. NVIDIA Rubin이 Blackwell(Expert당 1개) → Rubin(전체 공유)으로 바꾼 방식과 동일. MoE의 Expert 구조가 실제 HW ISA에 직접 반영. 신호 ↗ 07-22
MoE · EP
MoE 사전학습 세계기록 — GB300 NVL72로 프론티어 MoE 학습 실증. Expert 분할·EP가 대규모 스케일업에서 작동함을 보인 사례. 신호 ↗ 07-21
Attention · GQA
Kimi K3 KDA 선형 어텐션 — KV 전송 대역폭을 최대 10배 절감. GQA가 KV Head를 줄여 메모리를 아끼던 계보를, 선형 어텐션으로 더 밀어붙인 것. 신호 ↗ 07-19
TP · EP 규모
2.8T 대형 모델 추론엔 NVL72 — Kimi K3 2.8T는 FP4로 눌러도 단일 B200 불가, 노드 간 대역폭이 관건. TP·EP로 여러 GPU에 흩뿌려야만 서빙 가능한 규모 = 스케일업 도메인이 병렬 기법을 강제. 신호 ↗ 07-17
종합 · TCO
베라 루빈 NVL72 vs GB200 추론 분석 — 스케일업 도메인이 커질수록 TP·EP·MoE 효율이 perf/MW(전력당 성능)로 직결. 개념이 실제 TCO 숫자로 환산되는 지점. 뉴스레터 ↗ 07-23
