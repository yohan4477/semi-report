AI 모델은 학습과 추론으로 나뉜다 — 그 안에서 무엇을, 어떤 단위로 나누느냐가 GQA·TP·MoE·EP를 가른다

출발점은 두 국면 — 학습(Training)과 추론(Inference, 다시 Prefill·Decode). 국면마다 병목이 다르고, 그 병목을 푸는 게 모델을 나누는 기법들(구조적 GQA·MoE, 병렬 TP·EP)이다. 아래는 위에서 아래로 — 큰 국면 → 계층 구조 → 나누는 기법 → 물리·정밀도 → 한 장 통합.

1 · 큰 그림 — 학습 vs 추론, Prefill vs Decode

AI 모델을 쓰는 방식은 두 국면. 추론은 다시 두 단계로 쪼개진다. 어느 국면·단계냐에 따라 병목과 핵심 기법이 완전히 달라진다.

AI 모델 사용
compute-bound학습 (Training) — 순전파+역전파, 모든 토큰 병렬
추론 (Inference) — 두 단계로 쪼개짐

Prefill은 GPU를 꽉 채우는 연산, Decode는 대역폭에 굶주리는 순차 연산 → 그래서 KV 절감(GQA)·스케일업(NVL72)·Prefill 전용 가속기(CPX)가 갈린다. 이래서 요즘 추론을 Prefill/Decode로 분리(disaggregation)해 서로 다른 하드웨어에 태운다.

4 · 블록별 나누기 기법 — 병렬 배치 비교

1개 레이어 안에서 어느 블록을 어떤 단위로 나누는가에 따라 기술이 갈린다. 구조적 = 메모리·연산 절감, 병렬 = 여러 GPU 분산.

① Attention Block

Head 묶음 방식으로 나눔

MHA / GQA 구조적
MHA: Query·Key·Value Head를 1:1:1 보유. GQA: 여러 Query Head가 1개 K/V Head 그룹을 공유 → KV Cache 메모리 절감.
TP · Tensor Parallel 병렬
하나의 연산을 쪼개 여러 GPU가 동시 수행. Attention은 Head 단위로 분배 (예: 32 Head → 4 GPU가 8개씩).

② MLP / FFN Block

Expert 분할 방식으로 나눔

Dense 기존
모든 토큰이 단 하나의 거대한 MLP 가중치를 통과.
MoE · Mixture of Experts 구조적
MLP를 통째로 여러 개 복사해 독립 Expert로 분할. Router가 토큰별 1~2개만 선택 → 파라미터 용량↑, 실제 FLOPs는 절감.
EP · Expert Parallel 병렬
MoE의 각 Expert를 서로 다른 GPU에 분산 배치하는 병렬 기법.

7 · 물리 계층 — GPU → 랙 → 클러스터, 스케일업 vs 스케일아웃

병렬 기법은 결국 어떤 하드웨어 계층 위에 올리느냐로 갈린다. GPU가 트레이·랙·클러스터로 쌓이는데, NVLink로 묶인 랙(NVL72) 안은 스케일업, 랙과 랙 사이는 스케일아웃이다.

AI 팩토리 (데이터센터) — Pod 여러 개
스케일아웃Pod / 클러스터 — 랙 여러 개, Ethernet·InfiniBand 연결
스케일업 도메인Rack = NVL72 — 72 Rubin GPU + 36 Vera CPU, NVLink로 한 덩어리
└ Compute Tray — Vera CPU 1 + Rubin GPU 2 (NVLink-C2C, 액냉)
└ GPU (Rubin) — HBM4 288GB · 듀얼다이 1패키지 · NVLink 6 (3.6TB/s)

이 경계선(NVLink 랙 안 vs 밖)이 곧 병렬 기법이 갈리는 선 — 통신 잦은 TP·EP는 랙 안(스케일업), 덜한 DP·PP는 랙 밖(스케일아웃). GPU 자체 내부 구조는 GPU 구조 시뮬레이터 ↗.

스케일업 (Scale-up)

노드 내부 · 초고대역 링크로 GPU를 한 덩어리로

NVLink / NVSwitch · NVL72
Rubin NVLink 6 = 3.6TB/s. 통신 잦은 TP·EP가 여기. "대역폭 벽"을 노드 안에서 해결. NVL72가 이 도메인을 72 GPU로 키운 게 대형 추론의 열쇠.

스케일아웃 (Scale-out)

노드 간 · 이더넷/InfiniBand로 대수를 늘림

Spectrum-X · InfiniBand · RNG
통신 덜한 DP·PP가 여기. 클러스터 규모로 확장. 아마존 RNG 랜덤그래프 토폴로지처럼 오버서브스크립션·비용이 관건.

매핑: TP → 스케일업 · EP → 스케일업 우선 · DP·PP → 스케일아웃 · SP → 둘 다.

8 · 정밀도·양자화 — FP16 → FP4

비트를 줄일수록 메모리·대역폭·용량↓, 처리량↑. Decode(대역폭 병목)에 특히 효과. 학습은 BF16/FP8 혼합, 추론은 FP8/FP4 양자화가 표준.

FP32 기준→ BF16 학습 표준→ FP8 Hopper~→ FP4 NVFP4·MXFP · Blackwell·Rubin

품질 손실은 블록 스케일링(MXFP)·LUT 압축으로 방어. 단 비트를 줄여도 파라미터 용량은 남는다 — Kimi K3 2.8T는 FP4로 눌러도 단일 B200에 안 들어가 NVL72급이 필요하다. Rubin은 NVFP4로 50PFLOPS, LUT-B 압축까지 얹었다.

★ 종합 한 장 — 모든 기법 × 4개 축

위 전 섹션이 이 표 한 장으로 수렴한다. 4개 축이 직교 — 어느 블록(Attn/MLP) · 유형(구조적/병렬) · 핵심 단계(학습/Prefill/Decode) · 어느 네트워크(스케일업/아웃).

9 · 실제 신호에서 — 이 개념이 뉴스에 나타나는 곳

MoE
MI500X TDM MoE 디스크립터 공유 방식 — AMD가 전 Expert 공유 descriptor 도입. NVIDIA Rubin이 Blackwell(Expert당 1개) → Rubin(전체 공유)으로 바꾼 방식과 동일. MoE의 Expert 구조가 실제 HW ISA에 직접 반영. 신호 ↗ 07-22
MoE · 5D
MoE 사전학습 세계기록 — GB300 NVL72로 프론티어 MoE 학습 실증. Expert 분할·EP + DP·TP·PP가 대규모 스케일업에서 겹쳐 작동함을 보인 사례. 신호 ↗ 07-21
Prefill 전용
루빈 CPX 특화 가속기·랙 — NVIDIA가 추론을 Prefill/Decode로 분리하고, 컨텍스트 처리(Prefill) 전담 칩 CPX를 별도 출시. 단계 분리가 하드웨어 제품으로 실현된 사례. 뉴스레터 ↗ 25-09-10
Decode · GQA
Kimi K3 KDA 선형 어텐션 — KV 전송 대역폭을 최대 10배 절감. Decode가 KV Cache 읽기에 대역폭을 잡아먹는 문제를, GQA 계보를 선형 어텐션으로 더 밀어붙여 완화. 신호 ↗ 07-19
Decode · 양자화
2.8T 대형 모델 추론엔 NVL72 — Kimi K3 2.8T는 FP4로 눌러도 단일 B200 불가, 노드 간 대역폭이 관건. 양자화로도 안 줄는 용량 = TP·EP로 여러 GPU에 흩뿌려야만 서빙 가능한 규모. 신호 ↗ 07-17
Scale-up
NVLink = 스케일업 네트워크 — NVIDIA가 NVLink를 AI 팩토리용 스케일업 패브릭으로 정의(Rubin NVLink 6, 3.6TB/s). TP·EP 통신을 노드 안에서 감당하는 물리 기반. 신호 ↗ 07-20
Prefill+Decode
베라 루빈 NVL72 vs GB200 추론 분석 — 스케일업 도메인이 커질수록 TP·EP·MoE·양자화 효율이 perf/MW(전력당 성능)로 직결. 개념이 실제 TCO 숫자로 환산되는 지점. 뉴스레터 ↗ 07-23
