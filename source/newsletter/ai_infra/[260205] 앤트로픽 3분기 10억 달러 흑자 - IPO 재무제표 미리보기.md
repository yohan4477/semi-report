## 🔑 용어 정리

본문을 순서대로 읽기 전에 알아두면 좋은 용어들입니다. 자세한 수치와 설명은 본문에서 처음 등장하는 위치에 나옵니다.

- **ARR (Annual Recurring Revenue, 연환산매출)**: 지금 이 달의 매출 속도를 1년치로 환산한 값 — 실제 지난 1년 실적이 아니라 "지금 속도가 유지되면 연간 얼마"라는 스냅샷
- **NNARR (Net New ARR, 순증 연환산매출)**: 한 달 동안 새로 늘어난 ARR — 신규 고객 유치분과 기존 고객 사용량 증가분을 합친 월간 성장 속도
- **TaaS (Token-as-a-Service)**: 고객이 모델 개발사(랩)에서 직접 사지 않고 AWS·구글 같은 클라우드를 거쳐 토큰(모델 사용량)을 구매하는 간접 유통 방식
- **EBTIT (Earnings Before Training, Interest, and Taxes)**: 신규 모델 학습 비용을 빼기 전 이익 — AI 랩이 지금 얼마나 재투자할 실탄을 쥐고 있는지 보여주는 지표
- **GAAP EBIT (일반회계기준 영업이익)**: 미국 회계기준(GAAP)으로 계산한 영업이익 — 스톡옵션 등 비현금성 비용까지 반영해 기업이 자체 발표하는 비GAAP 지표보다 보수적
- **Net Dollar Retention (순매출유지율)**: 1년 전부터 있던 기존 고객들의 매출이 지금 몇 %로 늘거나 줄었는지 나타내는 비율 — 신규 고객 없이 기존 고객만으로 매출이 얼마나 느는지 측정
- **ARR per MW (MW당 매출)**: 컴퓨트(전력) 1메가와트를 굴려서 만들어내는 연환산매출 — 같은 하드웨어로 얼마나 돈을 잘 버는지 나타내는 효율 지표

## 1. 서론: 앤트로픽 IPO가 던지는 질문

**📌 핵심:**
- 앤트로픽은 2026년 6월 1일 비공개로 IPO를 신청 — 앤트로픽·OpenAI 합산 ARR(연환산매출)이 이미 약 1,000억 달러에 달하는 가운데, 두 회사 중 앤트로픽이 먼저 움직임
- 3분기(3Q26) 기준 앤트로픽은 GAAP 영업이익(EBIT) +10억 달러(매출 대비 6% 마진)로 흑자인 반면, OpenAI는 영업이익률 -100%로 여전히 대규모 적자 — 이 격차가 두 회사의 자금조달 우위를 가름
- OpenAI는 자체 IPO를 2027년으로 미룬다는 보도가 나오면서, 두 회사 모두 상장을 통해 자금을 조달할 능력이 있는지에 대한 의문이 제기됨
- 결론: 비공개 신청이라 공식 수치는 없지만, SemiAnalysis Tokenomics팀이 SKU·요금제·고객 유형별로 상향식(바텀업)으로 추정해온 재무모델을 최근 WSJ 보도가 정확성 면에서 뒷받침 — 이 문서는 그 추정 재무제표를 공개

## 2. IPO 배경 - 왜 지금 상장하는가

**📌 핵심:**
- 앤트로픽은 이미 1,000억 달러 넘게 조달했고 비GAAP 기준으로는 흑자인데도 상장을 택한 이유는 3가지 — ① 주식·채권 시장 접근(향후 필수) ② 감사된 공개 재무제표로 대형 계약 체결이 쉬워짐 ③ 직원의 지분 매각을 쉽게 해 인재 전쟁에서 우위
- 2030년까지 OpenAI·앤트로픽 합산 컴퓨트 수요가 100GW를 넘을 전망 — 2025년에 겨우 2.5GW, 2026년 5GW가 추가된 것과 비교하면 2030년까지 90GW 이상의 순증설이 필요한데, 현재 두 회사가 확보한 컴퓨트는 약 6GW뿐
- 1분기(1Q26) 동안 사용자들이 이 컴퓨트 부족을 체감 — 사용 제한(rate limit), 다운타임 등으로 Claude 이용이 어려워짐. 앤트로픽 API 사업의 매출총이익률이 이미 매우 높아, 공개시장 자금 조달로 이 마진을 레버리지 삼아 경쟁사가 따라오기 힘든 수준으로 컴퓨트 확보 경쟁에서 우위를 점할 수 있음
- 결론: 구글이 6월 848억 7,500만 달러를 증자하고 메타도 수백억 달러대 조달설이 도는 등 하이퍼스케일러조차 주식 발행에 나선 것은 AI 컴퓨트 수요가 그만큼 강력하다는 신호 — 앤트로픽 IPO는 이 자금조달 경쟁에 참여하려는 시도

## 3. 사업 구조 - 구독과 API 매출 믹스

**📌 핵심:**
- ChatGPT가 소비자에게 "아이폰의 순간"이었다면, Claude Code는 B2B(기업간거래)에서의 변곡점 — Claude Code는 이제 전체 GitHub 커밋의 7% 이상을 차지하며 2026년 초 앤트로픽 ARR 급가속을 이끔(2025년 말 90억 달러 → 현재 600억 달러 이상)
- 앤트로픽 매출의 75\~85%는 사용량 기반 API 매출, 구독은 15%에 불과 — 반대로 OpenAI는 역사적으로 소비자·구독 중심(1Q26 기준 65%+가 구독)이었으나 최근 변화 중
- 소비자 구독은 Free·Pro($20/월)·Max 5x($100/월)·Max 20x($200/월) 4단계, 팀(Team) 구독은 5\~150석 기업 대상 $20 또는 $100/월(연간 결제 시), 엔터프라이즈 구독은 별도 사용량 요금 없이 150석 초과 시 API 요금이 그대로 부과됨
- 결론: API 매출은 좌석(seat)이 아니라 토큰 단위로 과금되어 매출 상한이 없음 — 같은 고객이 에이전틱 워크플로우를 더 많이 쓸수록 신규 고객 없이도 매출이 계속 늘어나는 구조

📌 용어 풀이: 왜 "ChatGPT=소비자 아이폰 모먼트, Claude Code=B2B 변곡점"인가
> - ChatGPT는 일반 소비자가 처음으로 AI를 일상적으로 쓰기 시작한 계기였다면, Claude Code는 기업(개발팀)이 AI에 실제 업무를 맡기기 시작한 계기라는 의미

## 4. 성장 궤적과 손익 - ARR 폭증과 마진 개선

**📌 핵심:**
- Claude Code가 1분기(1Q26) 폭발적으로 흥행하며 앤트로픽 ARR을 2025년 말 90억 달러에서 단 한 분기 만에 300억 달러로 밀어올림(1월 +30억, 2월 +70억, 3월 +110억 달러) — 그중에서도 API 매출이 성장을 주도
- 매출총이익률은 2024년 -94%(원가가 매출보다 컸다는 의미)에서 현재 60%대 중반까지 상승 — 새 모델일수록 가격이 높아지고 칩 세대가 바뀔수록 토큰당 처리량이 늘어, 컴퓨트 원가는 고정인데 매출만 늘어나는 구조 때문
- 앤트로픽의 MW(전력 1메가와트)당 ARR은 9개월 전 1,600만 달러에서 올해 하반기 6,000만 달러 수준까지 상승 전망 — 반면 OpenAI는 무료 사용자 9억 명 이상을 이용자 1명당 월 약 0.70달러 원가로 서빙하며 전체 매출총이익률을 20\~30%p 깎아먹는 구조적 열위
- 결론: 3Q26 앤트로픽 GAAP 영업이익(EBIT)은 +10억 달러(마진 6%)로 흑자 전환, OpenAI는 여전히 -100% 마진 — 이 격차가 재투자 여력의 차이로 직결

---

### ARR 폭증 타임라인 - 90억에서 300억, 다시 600억 달러 이상으로

    B --> C["1Q26 말<br/>ARR 300억 달러"]

같은 기간 앤트로픽의 무료 소비자 월간활성이용자(MAU)는 약 5,500\~6,000만 명 수준이며 이 중 9%가 유료 전환한 것으로 추정됩니다. 반면 OpenAI는 연초부터 주간활성이용자(WAU) 약 9억 5,000만 명 선에 정체돼 있고, 유료 전환율도 약 6%로 앤트로픽보다 낮습니다.

📌 용어 풀이: 순증 ARR(Net New ARR)이 매출총액보다 중요한 이유
> - ARR 총액은 "지금 속도로 1년 가면 얼마"를 보여주지만, 순증(Net New) ARR은 "이번 달에 얼마나 더 늘었는지" 성장 속도 자체를 보여줌
> - 앤트로픽은 이 순증 속도가 월 100억 달러 이상까지 올라온 상태 — 신규 모델 출시가 이 속도를 한 단계 더 끌어올리는 촉매로 작용한다는 것이 저자들의 관찰

### 매출총이익률 반전 - 2024년 -94%에서 현재 60%대 중반으로

Claude가 아닌 랩 전체가 매출총이익률 개선의 수혜를 봤습니다. 신규 모델은 대체로 가격이 더 높게 책정되고, 서로 다른 가속기 칩은 토큰 처리량이 다르며 시간이 갈수록 추론에 맞게 최적화됩니다. 컴퓨트 원가는 컴퓨트 1단위당 거의 고정돼 있으므로, ① 같은 컴퓨트로 더 많은 토큰을 뽑거나 ② 그 토큰에 더 높은 가격을 매길 수 있으면 늘어난 매출의 거의 100%가 그대로 이익으로 흘러갑니다.

이 매출총이익 격차는 그대로 재투자 여력의 격차로 이어집니다. 매출총이익이 클수록 신규 모델 학습·연구에 더 많이 재투자할 수 있고, 이는 다음 모델의 ARR·매출총이익을 다시 끌어올리는 선순환을 만들어 경쟁사 대비 재투자 규모를 계속 벌리는 구조입니다.

매출총이익률은 앞으로도 계속 변할 수 있습니다. 앤트로픽은 최근 SpaceX와의 계약 사례처럼 시장가 2배 이상으로 신규 컴퓨트를 임대해주고 있고, AWS 베드락(Bedrock)처럼 20\~30%를 매출 분배로 지급해야 하는 채널의 비중도 늘고 있습니다. 하지만 API 매출총이익률이 80%대를 넘는 만큼, 앤트로픽은 현재 60%대 중반 수준을 당분간 유지하거나 오히려 개선할 여력이 충분합니다.

요약하면, 매출총이익률과 영업이익률의 뚜렷한 격차는 앤트로픽에게 OpenAI 대비 명확한 재투자 우위를 제공합니다. 이 현상은 뒤의 "컴퓨트와 장기 랩 경제학" 장에서 더 깊이 다룹니다. 앤트로픽은 집중력과 실행력을 갖추면 AI 랩도 훌륭하고 수익성 있는 사업이 될 수 있음을 보여주고 있으며, 저자들은 투자자들이 상장 시 다음 단계 컴퓨트 구축 자금을 넉넉히 공급하며 이를 보상할 것으로 예상합니다. OpenAI 입장에서는 초기 Codex·API 모멘텀을 재투자 가능한 마진으로 계속 전환하고, 구축 자금 조달을 위한 수익성 개선 경로를 제시하는 것이 관건입니다.

## 5. 코딩 시장과 대형 고객 집중도

- 오늘날 랩 매출은 소수의 "파워 유저"가 이끔 — 지출이 코딩 워크플로우에 좁고 두텁게 집중돼 있지만, 그 안에서도 폭넓은 저변이 있어 메타가 앤트로픽 전체 매출의 3\~5%에 불과함에도 최대 고객으로 추정됨

## 6. TaaS 사업 구조 - 하이퍼스케일러를 통한 간접 판매

**📌 핵심:**
- 토큰 서비스형(TaaS) 사업 모델이 최근 급성장 — 고객이 랩에서 직접 사지 않고 AWS 베드락(Bedrock), 애저 파운드리(Foundry), 구글 제미나이 엔터프라이즈(옛 버텍스) 같은 외부 클라우드 채널을 거쳐 모델을 구매
- 앤트로픽의 간접 채널(베드락·파운드리 등) 매출 비중은 1분기 전 5\~10%에서 현재 15\~20%로 빠르게 확대
- 대형 엔터프라이즈가 간접 채널을 선호하는 이유 3가지 — ① 특정 모델에 락인되지 않고 자유롭게 다른 모델로 전환 가능 ② 이미 확보한 약정 사용량(committed usage)·ELA를 그대로 활용 ③ 이미 신뢰하는 보안·컴플라이언스 체계, 6\~12개월짜리 신규 벤더 검증(RFP) 절차 불필요
- 결론: 2Q26 기준 TaaS 시장 규모는 280억 달러로 추정, 이 중 85%를 3대 클라우드(AWS·MS·구글)가 점유 — 20\~30% 매출 분배를 하이퍼스케일러에 내주더라도, 영업직군(BDR·기업영업·영업엔지니어·고객성공팀)을 자체 구축하는 것보다 저렴하다는 게 랩들의 계산

---

### 왜 대형 고객은 랩 직판보다 하이퍼스케일러 채널을 선호하나

```mermaid
flowchart TD
    Enterprise["대형 엔터프라이즈가<br/>간접 채널(TaaS) 선호하는 이유"] --> R1["① 모델 자유 전환<br/>(6개월 뒤 더 나은 모델<br/>나오면 쉽게 갈아탐)"]
    Enterprise --> R2["② 기존 약정 사용량·<br/>ELA 그대로 활용 가능"]
    Enterprise --> R3["③ 이미 신뢰하는 보안·<br/>컴플라이언스, 신규 벤더<br/>검증(RFP) 불필요"]
    style Enterprise fill:#eff6ff,stroke:#3b82f6
    style R1 fill:#f0fdf4,stroke:#16a34a
```

앤트로픽은 클로드를 베드락·파운드리·제미나이 엔터프라이즈 플랫폼(옛 버텍스)에서 모두 제공하며, 3대 하이퍼스케일러가 대형 기업이 선호하는 요소를 모두 갖추고 있습니다. 국제 워크로드의 경우 지연시간(latency)도 더 우수한데, 앤트로픽은 직접 API 요청은 전부 미국으로 라우팅하기 때문입니다.

### 간접 채널 매출 비중 - 5\~10%에서 15\~20%로 급성장

```mermaid
flowchart TD
    Before["1분기 전<br/>간접 채널 비중<br/>5~10%"] --> After["현재<br/>간접 채널 비중<br/>15~20%"]
    After --> Market["TaaS 시장 전체 규모<br/>280억 달러(2Q26 기준 추정)"]
    Market --> Share3["3대 클라우드<br/>(AWS·MS·구글)가<br/>85% 점유"]
    style After fill:#fff7ed,stroke:#ea580c,stroke-width:2px
    style Share3 fill:#f0fdf4,stroke:#16a34a
```

랩 입장에서도 이 구조가 나쁘지 않습니다. 고객에게 모델을 쉽게 바꿀 수 있는 자유를 주고 매출의 일부를 나눠주는 것이 손해처럼 보일 수 있지만, TaaS 파트너는 직접 서빙보다 유리한 IaaS(인프라 서비스) 수수료 조건으로 랩에 자본 유연성을 제공하고, 대형 엔터프라이즈·클라우드 고객 기반에 접근할 통로를 열어줍니다. 엔터프라이즈 영업은 원래 어려운 일이라, 연간 수천억 달러 규모의 기존 기업 고객 지출 기반을 가진 하이퍼스케일러를 통하는 편이 앤트로픽·OpenAI 모두에게 유리합니다. 많은 경우 자체 BDR·기업영업·영업엔지니어·고객성공팀을 꾸려 신규 고객 확보·확장·유지를 관리하는 것보다, 하이퍼스케일러에 20% 매출 분배를 내주는 편이 더 낫다는 것이 랩들의 계산입니다.

기업의 TaaS 채택은 이미 강한 편입니다. 이런 요인들 때문에 저자들은 TaaS가 앞으로도 강력한 사업 모델로 자리잡을 것이며, 앤트로픽 ARR에서 이런 간접 관계가 차지하는 비중이 계속 커질 것으로 봅니다.
