# LinkedIn 게시물 처리 규칙

SemiAnalysis LinkedIn 게시물(https://www.linkedin.com/company/semianalysis/posts/)을 수집·정리해 **주간 브리핑**과 **통합 리포트 최신 신호**로 가공하는 규칙. TRANSFORMATION_RULES.md·REPORT_RULES.md와 같은 단일 소스 원칙을 따른다.

## 1. 수집 방식

- **기본 방식 (CDP 자동 수집)**: 디버그 포트(9223)를 연 크롬을 띄우고 사용자가 로그인 → 게시물 페이지에서 **최신순 정렬** → 수집 스크립트가 로그인·피드를 자동 감지해 스크롤 후 HTML을 `input/linkedin_feed_raw.html`로 저장 (스크립트: 세션 scratchpad의 collect_linkedin.py, 크롬 실행 시 `--remote-debugging-port=9223 --user-data-dir=<임시 프로필>`, websocket은 `suppress_origin=True` 필수)
- 대안: 사용자가 직접 `Ctrl+S` 저장한 HTML을 `input/`에 넣거나, 게시물 텍스트를 채팅에 붙여넣기
- 파싱: HTML에서 게시물별 날짜·본문·(가능하면) 반응 수 추출. 날짜가 상대 표기("2w")면 저장 시점 기준으로 절대 날짜로 환산해 기록
- **증분 처리 (중요)**: 게시물 대장에 이미 기록된 게시물(URN 또는 날짜+본문 첫 줄로 식별)은 건너뛰고 **새 게시물만** 파싱 — 재수집 시 전체를 다시 처리하지 않음

## 2. 필터 (쓸모없는 게시물 제외)

**제외 대상**: 채용 공고, 행사·웨비나 홍보, 수상·기념 축하, 단순 구독 유도(내용 없는 뉴스레터 링크만 있는 것)
**포함 대상**: 시장·기술 분석 코멘트, 숫자·차트가 담긴 게시물, 뉴스레터 핵심 요약, 공급망·기업 동향 신호
- 제외한 게시물도 대장에 제목 한 줄 + `제외(사유)` 로 기록 — 재처리 방지용 (본문은 저장하지 않음)

## 3. 게시물 대장 — `링크드인/[YYMM] 게시물 대장.md` (월별)

게시물당 하나의 표 행:

| 날짜 | 요약 (1\~2줄, 한국어) | 핵심 숫자/신호 | 카테고리 | 연결 |
- **카테고리**: CATEGORIES.md의 유효 태그 재사용
- **연결**: 관련된 `원본 해석/` 문서나 `통합 리포트/`가 있으면 파일명 링크
- 핵심 숫자 중 문서 간 비교 가치가 있는 것은 `대장/숫자 대장.md`에도 append (출처를 "LinkedIn YYMMDD"로 표기)

## 4. 주간 브리핑 — `링크드인/브리핑/[YYMMDD] 주간 브리핑.md`

- 주 1회(또는 배치 처리 시 주 단위로 소급), **신호 5개 내외**, 경영진용 한국어
- 항목 형식: **한 줄 헤드라인 → 무엇이 달라지나(1\~2문장) → 출처 게시물 날짜**
- 분량 상한: 전체 1장(약 40줄) — 딥다이브가 아니라 훑어보기용
- 맨 위에 "이번 주 한 줄 요약" 1문장

## 5. 통합 리포트 "최신 신호" 연동 (핵심)

- 게시물이 기존 `통합 리포트/`의 관찰 포인트·핵심 판단과 관련되면, 해당 리포트에 **`## 최신 신호 (LinkedIn)`** 섹션(없으면 문서 하단에 신설)에 한 줄 추가:
  `- YYMMDD: 신호 내용 한 줄 (관련 관찰 포인트 명시)`
- 최신이 위로 오게 역순 정렬, 12개 초과 시 오래된 것부터 삭제
- 게시물 배치 처리와 같은 작업 안에서 수행 (두 번 읽지 않기 원칙)

## 5.5 대시보드 연동

- 배치 처리 후 **파이프라인 현황 Artifact**에 "최근 신호 TOP 5" 카드를 갱신 (게시물 날짜 + 한 줄 헤드라인) — 사용자가 모바일에서 브리핑 요약까지 한 페이지로 확인
- 주간 브리핑 전문은 별도 Artifact로 발행 가능 (요청 시)
- 전력 대시보드 등 기존 대시보드에는 해당 카테고리 신호만 선별 반영

## 6. 처리 파이프라인

1. `input/`의 HTML 파싱 → 게시물 추출
2. 필터 적용 → 대장 기록 (제외분 포함)
3. 숫자 대장 append
4. 주간 브리핑 작성 (주 단위 그룹)
5. 통합 리포트 최신 신호 갱신
6. 처리 완료한 HTML은 `input/처리완료/`로 이동, 커밋
