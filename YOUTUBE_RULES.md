# YouTube 자료 처리 계획·규칙 (v0.1 — 계획 단계, 미착수)

SemiAnalysis의 YouTube 두 채널을 자료원으로 추가하기 위한 계획. **아직 처리 시작 전** — 이 문서는 나중 세션이 바로 착수할 수 있게 남기는 설계도다.

## 채널 특성

- **@SemiAnalysis** (본채널): https://www.youtube.com/@SemiAnalysis
- **@SemiAnalysisWeekly**: https://www.youtube.com/@SemiAnalysisWeekly
- 성격: 뉴스레터(웹페이지) 내용의 **자세한 설명·부가 해설** — 글보다 쉽게 풀어 말하는 경우가 많음
- **중복 주의**: 두 채널이 영상을 공유해 같은 영상이 양쪽에 올라옴 → 영상 ID/제목 대조로 중복 제거, 본채널(@SemiAnalysis) 기준 우선

## 수집 방법 (로그인 불필요 — LinkedIn보다 자동화 쉬움)

1. `yt-dlp`로 채널별 영상 목록 + 메타데이터(제목·날짜·길이·설명) 추출: `yt-dlp --flat-playlist --print "%(id)s|%(upload_date)s|%(title)s" <채널URL>/videos`
2. 영상 다운로드 없이 **자막만** 추출: `yt-dlp --write-auto-sub --skip-download` (수동 자막 있으면 우선)
3. 두 채널 목록을 합쳐 영상 ID로 중복 제거 → `유튜브/영상 목록.md` 대장화 (ID | 날짜 | 제목 | 채널 | 처리 상태)
4. 증분 원칙: 대장에 있는 ID는 건너뜀 (LinkedIn과 동일)

## 활용 방안 (우선순위순)

1. **변환 문서 보충 (핵심)**: 뉴스레터 변환 시 이해가 어려운 대목을 같은 주제 영상 자막으로 보충 — 영상 해설이 더 평이해서, 우리 문서의 "쉬운 설명" 품질을 올리는 원천. 관련 영상이 있으면 변환 문서 헤더에 `> **관련 영상**: [제목](링크)` 병기
2. **미커버 주제 신호**: 뉴스레터로 안 나온 영상 단독 인사이트 → 요약해 게시물 대장과 같은 방식으로 축적, 통합 리포트 "최신 신호"에 반영
3. **주간 브리핑 통합**: LinkedIn 신호 + 그 주 영상 요약을 한 브리핑으로 (LINKEDIN_RULES.md 4절 형식 공유)
4. **숫자·예측 추출**: 자막에서 v17.0 대장 규칙과 동일하게 예측·핵심 숫자 추출

## 처리 파이프라인 (착수 시)

1. yt-dlp 설치 확인 → 두 채널 목록 수집 → 중복 제거 → 영상 목록 대장 커밋
2. 최근 영상부터 자막 추출 → sonnet 에이전트가 영상당 요약(3~5줄) + 관련 문서 매핑
3. 기존 변환 완료 문서와 매핑되는 영상부터 우선 처리 (보충 가치 즉시 발생)
4. 이후 신규 영상은 주간 배치로 (LinkedIn 수집과 같은 주기)
