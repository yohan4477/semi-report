# SemiAnalysis 카테고리 통합 리포트 규칙

카테고리 하나(예: `ai-infra/power`)에 걸친 여러 `원본 해석/` 문서를 하나로 종합하는 통합 리포트 작성 규칙입니다. 개별 문서 변환 규칙은 TRANSFORMATION_RULES.md, 카테고리 정의는 CATEGORIES.md를 참고하세요. 이 문서는 그 위에서 "여러 문서를 어떻게 하나로 합치는가"만 다룹니다.

## 목표 독자와 목적 (중요)

**독자**: 개별 문서 변환과 동일 — 전력·인프라 관련 투자·조달·전략 결정을 내려야 하는 경영진·임원·투자자 (TRANSFORMATION_RULES.md의 "목표 독자" 참고)

**목적**: **"SemiAnalysis 예측이 맞았는지 틀렸는지 채점"이 아님.** 통합 리포트는 애널리스트가 과거 예측 성적을 평가하는 문서가 아니라, 의사결정자가 "이 주제가 지금 어느 방향으로 얼마나 빠르게 움직이고 있고, 그래서 현재 전망을 얼마나 확신하고 베팅해도 되는지"를 빠르게 파악하도록 돕는 문서입니다.
- ❌ 틀린 프레이밍: "2024년 문서는 X라고 했는데 틀렸고, 2026년 문서가 Y로 정정했다" (성적표 채점)
- ✅ 맞는 프레이밍: "X 방향으로 흐름이 이어지고 있으며, 최근 문서가 이를 더 구체적인 수치로 재확인/가속화했다 → 이 방향에 대한 확신도가 올라감" (의사결정 참고)
- 이 구분은 아래 "핵심 원칙"과 "타임라인 다이어그램 규칙"의 노드 텍스트·화살표 라벨 작성 시 그대로 적용

## 트리거

두 가지 방식으로 생성:

1. **온디맨드**: "전력 카테고리 리포트 만들어줘" 같은 명시적 요청 — 아직 리포트가 없는 카테고리도 이 방식으로 처음 생성
2. **신규 문서 처리에 자동 연동 (v1.2)**: 새 SemiAnalysis 문서를 처리해달라는 요청을 받아 `원본 해석/`에 추가할 때, 그 문서의 카테고리 중 **이미 통합 리포트가 존재하는 카테고리**가 있으면, 문서 처리와 같은 요청 안에서 해당 통합 리포트도 갱신 — 새 문서를 반영해 시계열 흐름 섹션을 업데이트(새 데이터포인트 추가, 최신 노드 갱신)하고 문서별 요약에 새 항목 추가
   - 아직 통합 리포트가 없는 카테고리면 자동 생성하지 않음 — 그건 온디맨드 요청으로만 (트리거 1)
   - 정기 자동 생성(주간/월간)은 여전히 하지 않음 — 문서 유입이 비정기적이라 고정 주기가 실제 흐름과 안 맞음

## 대상 문서 판단

요청 시점에 CATEGORIES.md 기준으로 해당 카테고리 태그가 붙은 `원본 해석/` 문서를 전부 스캔. 문서 하나가 여러 카테고리에 걸쳐 있으면, 관련된 모든 카테고리 리포트에 중복 포함.

## 핵심 원칙: 최신 문서 가중치 + 시계열 흐름

같은 주제에 대해 여러 문서가 서로 다른 시점의 수치·전망을 담고 있을 때:

1. **최신 문서가 "현재 유효한 판단"**: 문서 간 수치·전망이 다르면, 발행일이 가장 최근인 문서의 내용을 지금 의사결정의 기준점으로 표시. 오래된 문서는 틀린 게 아니라 "그 시점 기준 판단"이자 지금 흐름을 이해하는 배경으로 취급
2. **흐름 자체가 확신도 신호**: 단순히 최신 값만 보여주지 않고, 시간에 따라 방향이 어떻게 움직였는지 타임라인으로 표시 — **같은 방향이 반복·가속되고 있다면 확신도를 높여도 되는 신호**이고, 방향이 뒤집혔다면 아직 불확실하다는 신호. "무엇이 맞았고 틀렸나"가 아니라 "이 방향에 얼마나 베팅해도 되는가"를 보여주는 것이 목적
3. **적용 대상**: 여러 문서에서 반복 등장하는 주제·수치에만 적용 (예: "미국 데이터센터 전력 수요 전망"은 5개 문서 중 3개에서 다른 시점 수치로 등장 → 타임라인 대상). 한 문서에만 나온 내용은 그 문서 요약으로만 취급

## 확신도 등급

시계열 주제마다 소제목 아래에 `**확신도: 높음/중간/낮음** — 근거 한 줄` 형식으로 표기한다. 판정 기준:

- **높음**: 2개 이상 문서가 같은 방향을 재확인했고, 최신 데이터포인트가 약 1년 이내
- **중간**: 단일 문서 기반이지만 정량 근거가 구체적인 경우, 또는 복수 문서지만 최신 데이터포인트가 오래된 경우
- **낮음**: 방향이 문서 간에 뒤집혔거나 근거가 정성적 — 방향 뒤집힘은 숨기지 말고 타임라인에 그대로 표시 (그 자체가 "아직 베팅하기 이르다"는 의사결정 신호)

## 다음 확인 포인트 규칙

원문 문서들에 실제로 언급된 예정 이벤트·지표만 쓴다 (예: 용량 경매 결과 발표, 규격 확정, 수주 공개, 제품 출하 시점). 리포트 작성자가 임의로 창작한 이벤트 금지. 각 항목에 영향받는 시계열 주제 번호(§1.N)와 방향(확신도 강화/약화 조건)을 표기.

## 문서 1개 카테고리 처리

온디맨드 요청 시 대상 문서가 1개뿐이어도 리포트 생성은 허용하되, "📌 현재 종합 판단"과 시계열 섹션은 넣지 않고 "문서별 요약 + 시계열 대기 안내" 형태로만 작성 (현행 냉각 리포트 방식을 공식화). 종합 판단·확신도는 문서 2개부터 의미가 있음.

## 구조

```markdown
---
categories: [ai-infra/power]
---

# [카테고리명] 통합 리포트

> **생성일**: YYYY-MM-DD (최초 생성 시점, 파일명에 고정된 날짜와 동일 — 이후 갱신되어도 안 바뀜)
> **최종 갱신일**: YYYY-MM-DD (트리거 2로 새 문서가 추가될 때마다 갱신)
> **대상 문서**: N개 (목록 + 발행일)

## 📌 현재 종합 판단

(3\~5개 불릿 + 결론 한 줄 — 이 카테고리가 지금 어느 방향으로 움직이고 있고, 어디에 얼마나 확신을 갖고 베팅해도 되는지. 경영진이 이 섹션만 읽어도 되도록 작성. 각 불릿에 관련 시계열 주제의 확신도를 함께 표기)

## 📑 목차

## 1. 시계열 흐름: 반복 등장 주제

(주제별로 하나씩, 발행일순 mermaid 순차 체인 다이어그램 — 최신 노드를 색상으로 강조. 각 주제 소제목 바로 아래에 확신도 한 줄 표기)

## 2. 다음 확인 포인트

(어떤 예정 이벤트·지표가 나오면 현재 판단이 강화/약화되는지 목록 — 각 항목이 어느 시계열 주제(§)에 영향을 주는지 표기)

## 3. 문서별 요약

(각 문서 1문단 요약 + 원본 링크)
```

## 타임라인 다이어그램 규칙

- TRANSFORMATION_RULES.md의 다이어그램 규칙(TD 방향, 형제 노드 최대 3개, 신호등 이모지 금지, 물결표 이스케이프)을 그대로 따름
- 시계열은 본질적으로 순차 관계이므로 `flowchart TD`의 순차 체인(A→B→C)으로 표현 — 각 노드는 "발행일 + 그 시점 전망"
- **최신 노드는 색상으로 강조** (예: `fill:#f0fdf4,stroke:#16a34a,stroke-width:2px`), 과거 노드는 중립색(`fill:#eff6ff,stroke:#3b82f6`)으로 구분 — "[현재 판단]" 같은 라벨은 괜찮지만 "[예측 적중]"·"[전망 실패]" 식 채점성 표현은 지양
- 노드 텍스트에 "왜 바뀌었는지"가 원문에 있으면 화살표 라벨로 표기 (예: `A -->|"ERCOT Batch Zero 제도 신설"| B`)
- 방향이 반복·가속되는 흐름은 화살표 라벨에 확신도로 표현 (예: `A -->|"6개월 뒤 같은 방향으로 재확인·가속"| B`)

## 출력 위치 및 파일명

`통합 리포트/[YYMMDD 생성일] [카테고리명] 통합 리포트.md` — **파일명의 날짜는 최초 생성일로 고정**. 트리거 2로 나중에 내용이 갱신돼도 파일명은 그대로 두고 같은 파일을 수정 (새 파일을 또 만들지 않음). 갱신 이력은 git 커밋과 "최종 갱신일" 필드로 추적.

- v1.1 (2026-07-04): 목표 독자·목적 섹션 명시, "예측 채점" 프레이밍 금지

  - 타임라인 다이어그램의 라벨·톤도 채점형("예측 적중") 대신 방향·확신도형("재확인·가속")으로 조정
- v1.2 (2026-07-05): 신규 문서 처리에 통합 리포트 갱신을 자동 연동
  - 배경: 사용자가 "새 문서 처리 요청 시 해당 카테고리에 통합 리포트가 있으면 자동으로 갱신하자"고 요청
  - 이미 리포트가 있는 카테고리에 새 문서가 추가되면, 그 리포트도 같은 요청 안에서 갱신 (파일명은 그대로, 내용만 갱신 — "최종 갱신일" 필드 추가)
  - 아직 리포트가 없는 카테고리는 자동 생성 대상이 아님 — 온디맨드 요청으로만 최초 생성
- v2.0 (2026-07-05): 경영진 요약·확신도 등급·다음 확인 포인트 섹션 신설, 문서 1개 카테고리 처리 공식화
  - 배경: 기존 리포트가 시계열 다이어그램+문서 요약만으로 구성되어, 정작 목적("어디에 얼마나 베팅해도 되는가")에 대한 종합 답이 없었음. 확신도를 명시적 등급으로, 판단을 바꿀 트리거를 "다음 확인 포인트"로 구조화
  - 문서 2개부터는 종합 판단·확신도·다음 확인 포인트 포함, 문서 1개 카테고리는 문서별 요약만으로 작성 (냉각 리포트 방식 공식화)
